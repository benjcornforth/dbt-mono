## How to Define Models (No SQL Required)

Edit `dbt/models.yml` using this format:

### Simple staging model (reads from a source)

```yaml
models:
  stg_customers:
    description: "Staged customer data"
    source: raw_customers
    columns:
      customer_id: { type: int, required: true, unique: true }
      email:       { type: string }
      signup_date: { type: date, cast: true }
```

### Model with data cleaning (quarantine bad rows)

```yaml
  customer_clean:
    description: "Clean customers"
    source: stg_customers
    materialized: table
    quarantine: "email IS NULL OR revenue < 0"
    columns:
      customer_id: { type: int, required: true }
      email:       { type: string, required: true }
```

### Join two sources

```yaml
  customer_orders:
    description: "Customers with their orders"
    sources:
      c: customer_clean
      o: stg_orders
    join: "c.customer_id = o.customer_id"
    materialized: table
    columns:
      customer_id: { from: c }
      order_id:    { from: o, required: true, unique: true }
      product:     { from: o }
```

### Aggregation (summary / gold layer)

```yaml
  customer_summary:
    description: "Customer metrics"
    source: customer_orders
    materialized: table
    group_by: [customer_id, email]
    columns:
      customer_id:   { type: int, required: true, unique: true }
      total_orders:  { expr: "count(order_id)", type: int }
      total_revenue: { expr: "sum(line_total)", type: decimal(10,2) }
```

## How to Define UDFs (Reusable Functions)

Add a `udfs:` block in `dbt/ddl/00_udfs.yml`:

### SQL UDF

```yaml
udfs:
  loyalty_tier:
    language: sql
    returns: string
    params:
      - { name: revenue, type: \"decimal(18,2)\" }
    body: |
      CASE
        WHEN revenue >= 1000 THEN 'GOLD'
        WHEN revenue >=  500 THEN 'SILVER'
        ELSE 'BRONZE'
      END
```

### Python UDF

```yaml
udfs:
  clean_email:
    language: python
    returns: string
    runtime_version: '3.11'
    handler: clean
    params:
      - { name: raw_email, type: string }
    body: |
      def clean(raw_email):
          if raw_email is None:
              return None
          return raw_email.strip().lower()
```

### Pandas UDF (vectorized — fast on large batches)

```yaml
udfs:
  average_score:
    language: pandas
    returns: double
    runtime_version: '3.11'
    handler: compute
    packages: [pandas, numpy]
    params:
      - { name: score_a, type: double }
      - { name: score_b, type: double }
    body: |
      import pandas as pd
      def compute(score_a: pd.Series, score_b: pd.Series) -> pd.Series:
          return (score_a + score_b) / 2.0
```

### Use in a column

```yaml
models:
  customer_summary:
    columns:
      tier: { udf: "loyalty_tier(total_revenue)", description: "Loyalty tier" }
```

## Broadcast Hints (Join Optimisation)

On Databricks, broadcasting a small table makes joins fly.
Add one line to any join model:

```yaml
models:
  customer_orders:
    sources:
      c: customer_clean    # small table
      o: stg_orders        # big table
    join: "c.customer_id = o.customer_id"
    broadcast: c            # ← one line
```

Generates: `SELECT /*+ BROADCAST(c) */ ...`

Multiple tables: `broadcast: [c, small_lookup]`

## How to Make Changes (Migrations)

Create a YAML file in `dbt/migrations/`:

```yaml
# dbt/migrations/001_add_loyalty_tier.yml
migration: "001_add_loyalty_tier"
description: "Add loyalty tier to customer summary"
changes:
  - model: customer_summary
    add_columns:
      loyalty_tier:
        type: string
        expr: "case when total_revenue > 1000 then 'gold' else 'silver' end"
```

Then run: `forge migrate`
